# 工作流 3：学情诊断

## 触发方式
老师提供某位或多位学生的多次作业记录

## 流程

```
[用户输入：学生姓名 + 历次作业成绩及表现]
       │
       ▼
[个体分析] → 每个学生的知识点掌握度评估
       │
       ▼
[班级聚合] → 找出共性问题、薄弱模块
       │
       ▼
[输出] → 个人诊断报告 + 班级共性问题 + 补救建议
```

## 知识点掌握评估维度

根据 Linux 知识体系 12 个模块，评估每个学生的掌握程度：
- 🟢 熟练：多次作业该知识点均正确
- 🟡 一般：偶尔出错，基本理解
- 🟠 薄弱：频繁出错，概念不清楚
- 🔴 未掌握：基本不会，需要从头教

## 输出模板

```
## 学情诊断报告

### 学生：张三

| 知识模块 | 掌握程度 | 说明 |
| Shell 脚本-变量 | 🟡 一般 | 变量赋值偶尔加空格 |
| Shell 脚本-条件判断 | 🟠 薄弱 | [ ] 内经常忘记空格 |
| 文件权限 | 🟢 熟练 | 正确使用 chmod 和 chown |
...

### 班级共性问题
1. **变量使用不规范** — 60% 学生存在变量赋值空格、不加引号问题
2. **管道理解不深** — 40% 学生不会用管道串联命令

### 建议补救措施
1. 下节课花 15 分钟专项练习变量赋值和引号使用
2. 安排一次管道与文本处理专项实验
```
